Key Design Decisions

Route placement: Both pages in (auth) route group using AuthLayout — consistent with login/register
SDK calls: sdk.auth.resetPassword() for request, sdk.auth.updateProvider() for completion (token in Authorization header, Medusa v2.6+)
Security: Always show "If an account exists..." message (prevents email enumeration). Email normalized to lowercase before SDK call.
Error states: Expired token, invalid token, password too short, password mismatch — all surface as inline errors
Client vs Server: Forms are client components (need useState for success state + password match). Pages are server components wrapping forms in AuthLayout.
