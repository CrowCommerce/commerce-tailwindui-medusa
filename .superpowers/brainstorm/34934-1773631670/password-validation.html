Password Complexity Validation

Shared validation — 8+ characters, enforced at every password entry point

Validation Rule

Minimum 8 characters
NIST SP 800-63B aligned — length over complexity

Shared Validation Function

lib/validation.ts (new file)
/** Minimum password length (NIST SP 800-63B) */
const MIN_PASSWORD_LENGTH = 8

export function validatePassword(password: string): string | null {
  if (password.length < MIN_PASSWORD_LENGTH) {
    return `Password must be at least ${MIN_PASSWORD_LENGTH} characters`
  }
  return null
}

Returns error message string or null. Used by both server actions and client components. Single source of truth — change the rule here, it updates everywhere.

Where It's Enforced

Server
signup() in customer.ts
Call validatePassword() before SDK register call. Return error string if invalid.
Server
resetPassword() server action (new)
Validate new password in the reset completion form's server action before SDK call.
Client
RegisterForm component
HTML minLength={8} attribute on password input + hint text below field.
Client
ResetPasswordForm component (new)
HTML minLength={8} on both password fields + hint text. Also client-side mismatch check for confirmation field.

NOT Applied To

login() — intentionally skipped
Existing accounts may have short passwords set before this rule existed. Validating on login would lock them out. They can set a stronger password via the reset flow.
